Retrieval-Augmented Generation (RAG)

RAG is a technique that combines information retrieval with large language model generation.
Instead of relying solely on the model's training data, RAG retrieves relevant documents from
a knowledge base and includes them as context when generating an answer.

A typical RAG pipeline has four stages:

1. Ingestion — Load documents (PDF, text, markdown), split them into chunks, and embed each chunk.
2. Indexing — Store embeddings in a vector database such as ChromaDB for fast similarity search.
3. Retrieval — When a user asks a question, embed the query and find the most similar chunks.
4. Generation — Pass the retrieved chunks and the question to an LLM to produce a grounded answer.

RAG reduces hallucinations by grounding responses in real source material and makes it easy to
update knowledge without retraining the model — just add new documents to the index.
